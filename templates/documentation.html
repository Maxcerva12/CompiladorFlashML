Documentación de FlashML

Introducción

FlashML es un lenguaje de marcado inspirado en la velocidad y dinamismo de *The Flash*. Diseñado para crear páginas web de manera rápida y estructurada, FlashML utiliza etiquetas temáticas como @velocista, @temporada, y @personaje que se transforman en HTML semántico. El compilador FlashML analiza el código, valida su estructura, y genera páginas web con estilos inspirados en el universo de *The Flash*.

Instalación y Uso

Para usar el compilador FlashML:

  1. Accede a la interfaz web en Compilador FlashML.
  2. Escribe tu código FlashML en el editor integrado (basado en CodeMirror).
  3. Haz clic en el botón de compilación (ícono de rayo) para procesar el código.
  4. Si la compilación es exitosa, usa los botones "Ver HTML" o "Descargar HTML" en el panel "Archivos Generados".
  5. Revisa los errores en el panel "Estado de Compilación" si ocurre algún problema.

Usa Ctrl+S para auto-indentar tu código y mantenerlo organizado.

Sintaxis Básica

Un documento FlashML debe comenzar con una etiqueta raíz @velocista. Las etiquetas se definen con @nombre, atributos como nombre="valor", y se cierran con @/nombre. Los comentarios se escriben con ##texto##. 

@velocista
  @titulo Central City @/titulo
  @temporada numero="1"
    @episodio
      @escena
        @personaje nombre="Barry Allen"
          @dialogo ¡Corre, Barry, corre! @/dialogo
        @/personaje
      @/escena
    @/episodio
  @/temporada
@/velocista

Este código genera una página HTML con un título, una sección de temporada, y un episodio con una escena y diálogo.

Reglas Semánticas

FlashML enforces strict semantic rules to ensure valid document structure:

  • Un documento debe tener al menos una etiqueta raíz @velocista.
  • Algunas etiquetas requieren atributos específicos (e.g., nombre para @personaje).
  • Las etiquetas solo pueden anidarse en contextos permitidos (e.g., @villano solo dentro de @villanos).
  • Atributos no permitidos generan errores semánticos.

Consulta la tabla de etiquetas para detalles sobre dónde puede usarse cada etiqueta y sus requisitos.

Ejemplo Completo

Este ejemplo muestra una página FlashML que describe un episodio de *The Flash* con personajes, diálogos, acciones, y villanos. 

@velocista
  @titulo Batalla en Central City @/titulo
  @temporada numero="2"
    @episodio
      @escena
        @lugar nombre="Central City"
          @personaje nombre="Barry Allen" actor="Grant Gustin"
            @dialogo
              ¡@superspeed Nadie puede detenerme! @/superspeed
            @/dialogo
            @accion velocidad="máxima"
              Barry usa @speedforce la Fuerza de Velocidad @/speedforce para esquivar un ataque.
            @/accion
          @/personaje
          @personaje nombre="Caitlin Snow" actor="Danielle Panabaker"
            @dialogo
              ¡Cuidado con @cryokinesis el hielo! @/cryokinesis
            @/dialogo
          @/personaje
        @/lugar
      @/escena
      @villanos
        @villano nombre="Captain Cold" poder="Criocinesis"
          Leonard Snart apunta con su arma de hielo.
        @/villano
        @villano nombre="Heat Wave" poder="Piroquinesis"
          Mick Rory lanza una ráfaga de fuego.
        @/villano
      @/villanos
      @imagen src="/images/battle.jpg" alt="Batalla en Central City" @/imagen
    @/episodio
  @/temporada
  ## Comentario: Escena épica de batalla ##
@/velocista

Este código genera una página HTML con una estructura semántica, estilos temáticos, y comentarios preservados como comentarios HTML.

Estilos Generados

El compilador FlashML incluye estilos CSS predefinidos inspirados en *The Flash*. Cada etiqueta tiene una clase CSS correspondiente que aplica estilos visuales:

  • .velocista: Borde rojo y fondo gris oscuro.
  • .titulo: Texto rojo con borde amarillo inferior.
  • .superspeed: Texto amarillo con estilo cursivo.
  • .villano: Fondo rojo oscuro para villanos.

Consulta el archivo generator.py para la lista completa de estilos.

Errores Comunes

Aquí hay algunos errores comunes y cómo evitarlos:

  • Etiqueta de cierre faltante: Asegúrate de cerrar cada etiqueta con @/nombre. 
    @velocista
  @titulo Sin cerrar
  ## Error: Falta @/titulo ##
@/velocista
  • Atributo requerido faltante: Por ejemplo, @personaje requiere nombre. 
    @personaje  ## Error: Falta nombre ##
  @dialogo Hola @/dialogo
@/personaje
  • Etiqueta en contexto inválido: Por ejemplo, @villano solo puede estar dentro de @villanos. 
    @velocista
  @villano nombre="Zoom"  ## Error: @villano no permitido aquí ##
@/velocista
  • Sintaxis inválida: Asegúrate de que los atributos estén entre comillas y que no haya espacios extra. 
    @velocista tipo = "página"  ## Error: Espacio antes de = ##
  ¡Hola!
@/velocista
  • Valor de atributo suelto: Un valor debe estar precedido por un nombre de atributo. 
    @velocista "página"  ## Error: Valor sin nombre de atributo ##
  ¡Hola!
@/velocista

Consejos Avanzados

Maximiza tu uso de FlashML con estos consejos:

  • Usa comentarios (##texto##) para documentar escenas complejas.
  • Aprovecha etiquetas de poder como @superspeed y @speedforce para resaltar acciones dinámicas.
  • Organiza tu contenido con @temporada y @episodio para una estructura clara.
  • Revisa el panel "Estado de Compilación" para depurar errores rápidamente.
